# Проверочный лист безопасности бизнес-систем PropDevelopment

## 1. Идентификация систем и анализ с точки зрения безопасности

В ходе анализа архитектуры PropDevelopment были выделены ключевые бизнес-системы, которые обрабатывают публичные, внутренние, конфиденциальные и секретные данные.

### Группа сервисов для продаж

К этой группе относятся витрина продаж, client-tour-app, client-mart-app, client-crm-app и client-mart-estate-app.

Системы обрабатывают данные клиентов, данные бронирований, данные онлайн-туров, данные сделок, сведения об объектах недвижимости и данные аутентификации покупателей.

С точки зрения безопасности эта группа является одной из наиболее критичных, так как через неё проходят персональные данные клиентов, юридически значимые данные сделок и данные, связанные с покупкой недвижимости. Основные риски: утечка персональных данных, искажение данных сделки, нарушение разграничения доступа и отображение данных одного клиента другому.

### Группа сервисов ЖКУ

К этой группе относятся витрина для собственников, tenant-core-app, CRM по собственникам и интеграции с поставщиками ресурсов ЖКХ и платёжными системами.

Системы обрабатывают данные собственников, данные по услугам ЖКХ, платежи, обращения, данные о ремонтах и данные интеграций с внешними партнёрами.

С точки зрения безопасности эта группа критична из-за работы с персональными и платёжными данными собственников, а также из-за интеграций с внешними API. В описании системы указано, что партнёрские API не имеют единых политик безопасности и могут позволять доступ к данным других управляющих компаний.

### Финансовый домен

К финансовому домену относятся accountant-service-1, база данных финансового учёта и служба каталогов.

Системы обрабатывают бухгалтерские документы, финансовые данные, данные платежей и транзакций.

С точки зрения безопасности этот домен критичен, так как нарушение целостности или утечка финансовых данных может привести к финансовым потерям, ошибкам в отчётности и нарушению требований законодательства.

### Домен данных

К домену данных относятся хранилище данных, DWH, BI и отчётность.

Системы аккумулируют данные из разных бизнес-доменов и используются для аналитики, отчётности и будущего развития BI-, ML- и AI-сервисов.

С точки зрения безопасности этот домен требует особого контроля, так как в нём накапливаются большие объёмы данных из разных источников. При отсутствии Data Governance возможны неконтролируемый доступ, использование некачественных данных и нарушение требований к обработке персональных данных.

### Системы аутентификации и управления доступом

К этой группе относятся auth-service, Active Directory и механизмы авторизации пользователей и сотрудников.

Эти системы отвечают за аутентификацию пользователей, управление учётными записями и доступом к сервисам.

С точки зрения безопасности они являются критичными, так как компрометация учётных записей, токенов, паролей или прав доступа может привести к несанкционированному доступу к клиентским, финансовым и внутренним данным.

## 2. Обоснование выбора разделов проверочного листа

Для проверочного листа были выбраны следующие разделы:

1. Управление доступом.
2. Безопасность данных.
3. Защита инфраструктуры.
4. Управление инцидентами.

Выбор этих разделов связан с выявленными проблемами PropDevelopment:

- отсутствует системный контроль потоков данных между IT-продуктами;
- нет достаточных ограничений доступа к данным со стороны систем обработки данных;
- есть несколько точек регистрации клиентов без единой координации;
- в личном кабинете иногда отображаются данные другого клиента;
- партнёрские API не имеют единых политик безопасности;
- внешние партнёры потенциально могут видеть или изменять данные, к которым не должны иметь доступ;
- компания обрабатывает персональные, финансовые, платёжные и аутентификационные данные;
- компания планирует развивать BI-, ML- и AI-сервисы на базе накопленных данных;
- требуется проверить соответствие требованиям законодательства РФ.

Разделы, связанные с физической безопасностью и управлением оборудованием, не были включены в основной проверочный лист, так как в описании системы нет достаточной информации о серверных помещениях, рабочих станциях, оборудовании и физических средствах защиты.
